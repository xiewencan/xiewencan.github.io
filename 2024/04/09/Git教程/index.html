  • 变基

  1. 确保你的本地分支是最新的：

    git checkout your-branch git pull origin master

  2. 开始变基：

    git rebase origin/master

  3. 解决冲突（如果有的话）：

    • Git 会提示你解决冲突。
    • 手动编辑文件，解决冲突。
    • 使用 git add <file> 添加解决冲突后的文件。
    • 继续变基：git rebase --continue。

  4. 完成变基后，如果你需要推送更改到远程仓库：

    git push --force

记住，变基是一个强大的工具，但如果使用不当，可能会导致复杂的合并冲突和历史问题。

开发流程

  • 第一步：新建分支，每次开发新功能，都应该新建一个单独的分支
  • 第二步：提交分支commit，分支修改后，就可以提交commit了。

git status 命令，用来查看发生变动的文件。
git commit 命令的verbose参数，会列出 diff 的结果。

  • 第三步：撰写提交信息

    提交commit时，必须给出完整扼要的提交信息
    第一行是不超过50个字的提要，
    然后空一行，罗列出改动原因、主要变动、以及需要注意的问题。
    最后，提供对应的网址（比如Bug ticket）。

  • 第四步：与主干同步，分支的开发过程中，要经常与主干保持同步。

  • 第五步：合并commit，开始变基，解决冲突，
  • 第六步：推送到远程仓库
  • 第七步：发出Pull Request
